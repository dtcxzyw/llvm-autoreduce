Read `reproducers.md` for all reproducer files and `issue.md` for the bug report. Use bash to inspect files.

CRITICAL: Do not read or write any files outside the current working directory. All operations are confined to this directory. Never access /tmp, /home, /etc, /var, or any other system path.

1. Classify the bug type: `crash` (opt/llc/lli crash), `miscompilation` (wrong code), or `unrelated`
2. Identify the primary reproducer file name
3. Extract a literal substring from the crash output that uniquely identifies this crash (empty string for miscompilation). Do NOT use regex — produce a plain text fragment.
4. Determine the opt pipeline from the issue

Write your findings to `extract.json` in this exact format:
{{
  "bug_type": "crash",
  "reproducer_file": "filename.ll",
  "crash_pattern": "literal substring from crash output, or empty string",
